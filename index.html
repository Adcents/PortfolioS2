Chat POC

Ik ben bezig geweest met Node.js en Socket.io. Deze heb ik gemaakt doormiddel van een tutorial te volgen. Wat mijn doel bij deze POC was om voor het eerst kennis te maken met node.js en socket.io en te begrijpen wat het inhouden en wat ik er zelf mee kan in de toekomst. Dit als voorbereiding op aankomende semesters. Doormiddel van comments in de code laat ik ook zien dat ik snap wat de code inhoud.

Wat ik gemaakt heb is een chat systeem. Waarbij je op pageload gevraagd wordt voor een naam. Zodra je deze naam ingevuld hebt wordt je naar een chat doorgestuurd. Als je een nieuw tabblad opent wordt er opnieuw gevraagd voor een naam. Dit is de tweede user. Deze joint dezelfde chat als de andere gebruiker. Hier kunnen ze met elkaar chatten. De gebruiker ziet zijn eigen naam als “U” en de tegenspeler de gekozen gebruikersnaam op het begin.

Aangezien ik met een eigen live server werk wordt dit implementeren op de website een stuk lastiger, vandaar dat ik hieronder een filmpje heb. De code is te vinden op github, zie hiervoor het githud modal onder development.

Door deze POC te maken heb ik een idee gekregen van hoe het opzetten van een node.js server in zijn werking gaat en hoe je socket.io kan gebruiken. Ik snap nu dat socket.io een live server is en dat ik doormiddel van nodemon dingen kan sturen zonder dat ik de server hoef te refreshen. Verder heb ik geleerd van de code zodat ik zelf dit toe kan gaan passen in de toekomst.

Algemene Reflectie Dev

Een algemene reflectie over development in semester 2.

Ik heb het idee dat ik het afgelopen semester voldoende aan development heb kunnen werken. Ik merk echter wel dat het snel wegglipt tussen de andere vakken aangezien UEX en SCO een stuk meer toepasbaar zijn op de proftaak. En dan heb ik het over de kwantiteit dev opdrachten voor de proftaak.

Ik heb dit semester veel bijgeleerd over het programmeren met JS. Ik ga hier in aankomende semesters zeker mee door. Ik kan sinds dit semester JS code goed lezen, ik snap wat er staat en kan het toepassen op mijn eigen manier. Ook kan ik JS-code schrijven door mijzelf een opdracht te geven en deze stap voor stap uit te zoeken en uit te voeren.

Ik heb aangetoond in de gemaakte POC’s dat ik om kan gaan met feedback, dat ik deze kan verwerken en toepassen. Ook heb ik aangetoond dat ik de gemaakte code snap doormiddel van comments in de code zelf en uitleg onder de opdrachten. Ik heb aangetoond dat ik kennis op kan nemen van voorbeelden zoals tutorials en deze toepassen naar mijn eigen POC’s.

Door moeite met plannen ben ik later begonnen met delen van dev dan ik had gewild. Ik hoop dat alsnog de juist kwaliteit en kwantiteit te leveren voor het behalen van dev. Omdat ik JS snap en graag wil gaan toepassen op een semester 3/4 niveau. Zo houd ik voor mij de uitdaging en kom ik niet weer met uitstelgedrag.

Dit is ook een reflectiepunt voor volgend jaar. Vakere feedbackmomenten zijn zeer verstandig en een betere planning. Als ik dat op orde heb, ga ik volgend semester 3/4 zeker halen. Hier heb ik de volle vertrouwen in.

Customer Journey

Bij de typografie opdracht heb ik doormiddel experimenteren met fonts een beeld gekregen over het karakter en de vormen van verschillende typografie. Hierbij heb ik mij laten inspireren door typografie posters op het internet (pinterest) maar ook zelf dingen uitgeprobeerd. Hieronder zie je een 7 tal aan poster. Ik bespreek per poster wat ik er van vind, wat ik geprobeerd heb en hoe ik hierdoor op het eindresultaat ben gekomen.

Poster 1:

De linker poster is het voorbeeld en de rechter poster is mijn eigen poster. Mijn poster is gekomen uit een van mijn inspiraties van het internet. Wat ik heb geprobeerd is om de typografie poster na te maken tot op zekere hoogte en er zelf een kleine draai aan te geven. In dit geval is dat vooral de tekst en de vormen. Door twee driehoeken op de achtergrond een slagschaduw te geven heb ik diepte gecreëerd in deze poster. Ik heb hier het font goed tot zijn recht proberen te laten komen doordat ik de openingen in het font zichtbaar laat en niet opvul doormiddel van overlapping met de rode balken. Nu lijkt het alsof de letters op sommige delen meer verbonden zijn met elkaar.

Poster 2:

Deze poster wilde ik eerst als typografie poster gebruiken. Echter wat blijkt uit feedback, en waar ik mij dus ook bij aansluit is dit geen goede typografieposter. Dit komt doordat ik doormiddel van extra balken toe te voegen aan de uiteindes van letters een vervorming creëer. Dat is iets wat je absoluut niet wilt hebben bij het gebruik van typografie omdat dit heel het effect van de typografie zelf weg haalt. Alsnog is dit een mooie poster die wel gebruikt zou kunnen worden als bandposter.

Poster 3:

Deze poster is voortgekomen uit het experimenteren met overlappen van het font. Ik heb geprobeerd letters zo veel mogelijk en zo goed mogelijk in elkaar te laten overlopen. Mijn doel was dat de meeste letters nog enigszins leesbaar bleven maar dat het geheel samen een onherkenbaar geheel werd. Het heeft nu een beetje iets weg van een soort chinees teken. Daarmee is mijn doel goed gelukt. Ik vind dit dus een goede typografie poster aangezien ik de sterke kanten van het font benut en de letters in hun waarde laat.

Poster 4:

Deze poster is weer voorgekomen uit het experimenteren van font overlappingen. Echter wilde ik bij deze poster iets anders proberen. Ik wilde proberen een soort spiegeling te creëren in het midden van de poster. Zodat de linker en rechter kant gelijkwaardig zijn. Sommige letters zijn echter al gespiegeld als je ze door midden hakt. Hier heb ik gebruik van proberen te maken op de middenlijn. Zoals je kunt zien bij de M bovenaan, de H onderaan en een rare hoofdletter T in het midden.

Poster 5:

Bij deze typografie poster heb ik geprobeerd een less is more stijl aan te houden. De letter K en F liepen hierbij perfect mooi in elkaar over en kon ik zo netjes vertonen dat beide letters leesbaar bleven. De kleurverschillen maken dit extra duidelijk. Echter is deze less is more wel erg less voor een typografie poster. Daarom is dit een minder goede poster.

Poster 6:

Bij deze typografie poster heb ik geprobeerd iets herkenbaars te maken doormiddel van het font. Wat mij na wat experimenteren een goed beeld leek was dit. Na feedback van andere merk ik dat iedereen er iets anders in ziet. En dat is zeker nooit slecht. Mijn intentie was echter het beeld van een mannetje. Met als Q het hoofd bovenaan, de A en 2x V’s als borst en armen. En de W als benen. De kleurenvakken heb ik er bij gezet om de poster wat meer opvulling te geven.

Poster 7:

Bij deze poster is het erg moeilijk om te zien wat de letter is. En of er überhaupt wel een letter in zit. De letter die gebruikt is is de letter V. Deze is vervolgens omgedraaid en erboven op gezet. Denk aan het Renault logo. Wat jammer is bij deze poster is zoals hierboven besproken, het is te onduidelijk om te zien welke letter en dus ook wel font hier gebruikt is. Met een enkele letter weer te geven is het ook erg moeilijk om een heel font over te brengen. Dit maakt dit dus een minder sterke foto dan de meeste andere posters hierboven.

SPRINT X

Het idee:

Mijn idee was om een stukje augmented reality te gaan creëren met 3D models. Daar ben ik in mijn selfie week ook een beetje mee bezig geweest. Ik wilde het nu graag ook proberen met architectuur. Dus het leek mij leuk om een gebouw of verdieping the 3D modeleren en die vervolgens met augmented reality in de “echte wereld” te zetten.

Het onderzoek:

Ik ben gaan kijken wat gepaste software is voor het maken van 3D models. Toen kwam ik al snel uit bij Blender of Sketchup. Na verder onderzoek bleek dat Sketchup een makkelijkere werkwijze heeft, zeker voor mij aangezien ik daar al vaker mee gewerkt heb. En dat het ook makkelijk te exporteren is naar Unity. Aangezien ik Unity wilde gaan gebruiken voor mijn augmented reality heeft dat dus zijn pluspunten.

Het proces:

Na mijn onderzoek ben ik aan de gang gegaan met het 3D modeleren van mijn eigen woonkamer in Sketchup. Ik ben begonnen met het opzetten van de muren. Vervolgens heb ik er stap voor stap de deuren en ramen ingezet, vervolgens heb ik het meubilair er verder in gezet. Als laatst heb ik alle texturen aangepakt. Nadat ik dit gedaan had was mijn 3D model klaar. En kon ik het gaan exporten. Dit heb ik gedaan naar een .dae bestand. Vervolgens heb ik deze map met texturen en het .dae bestand in Unity gesleept.

In Unity terecht gekomen heb ik een target image aangemaakt met Vuforia. In die target image laad ik een image die ik in de Vuforia database heb gezet. Deze database lees trackpoints uit van de afbeelding. Zodra ik mijn target image en mijn 3D model aan elkaar link wordt het 3D model getraceerd d.m.v. de trackpoints van de afbeelding.

Het resultaat:

Het resultaat is dit geworden. Hieronder vind je mijn pitch voor de sprint x demo’s.

Conclusie:

Tijdens sprint X heb ik veel geleerd. Ik weet nu hoe ik een object vanuit Sketchup in Unity kan krijgen en ook toepassen. Verder heb ik geleerd hoe ik een 3D model teken in Sketchup. Ook heb ik geleerd hoe image tracking werkt en hoe ik een 3D model aan een target image vast zet.

Ik ben zeer tevreden over het resultaat en wil graag verder in de toekomst met soortgelijke projecten.
